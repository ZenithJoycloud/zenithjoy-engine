## PRD - 统一 Subagent Gate 机制

**需求来源**: 审查发现 Subagent 调用机制不统一，规则未正确挂载

**功能描述**:
1. 统一所有 Gate 命名：gate:prd, gate:dod, gate:qa, gate:audit, gate:test, gate:learning
2. 在 steps 文件中嵌入完整的审核规则（从 gates/*.md 复制）
3. 明确循环逻辑：FAIL → 修改 → 再审核 → 直到 PASS

**涉及文件**:
- skills/dev/steps/01-prd.md（修改：嵌入 gate:prd 完整规则）
- skills/dev/steps/04-dod.md（修改：嵌入 gate:dod + gate:qa 完整规则）
- skills/dev/steps/05-code.md（修改：嵌入 gate:audit 完整规则）
- skills/dev/steps/06-test.md（修改：嵌入 gate:test 完整规则）
- skills/dev/steps/10-learning.md（修改：嵌入 gate:learning 规则）
- skills/dev/SKILL.md（修改：更新命名）

**成功标准**:
- [ ] 所有 steps 文件使用统一的 gate:xxx 命名
- [ ] 每个 gate 的 Subagent prompt 包含完整审核规则
- [ ] 循环逻辑清晰：FAIL → 修改 → 再审核

**非目标**:
- 不修改 gates/*.md 规则文件本身
- 不修改 hooks 配置
